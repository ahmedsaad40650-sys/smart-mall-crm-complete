# 🚀 دليل البدء السريع - Smart Mall CRM

## 📋 المتطلبات الأساسية

قبل البدء، تأكد من توفر:

- ✅ Docker 20.10 أو أحدث
- ✅ Docker Compose 2.0 أو أحدث
- ✅ 4GB RAM على الأقل
- ✅ 10GB مساحة تخزين فارغة

---

## ⚡ التثبيت في 3 خطوات

### 1️⃣ تحميل المشروع

```bash
# تحميل المشروع
git clone https://github.com/your-repo/smart-mall-crm.git
cd smart-mall-crm
```

### 2️⃣ تشغيل النظام

```bash
# تشغيل سكريبت الإعداد التلقائي
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### 3️⃣ الوصول للنظام

افتح المتصفح واذهب إلى:
```
http://localhost
```

استخدم بيانات الدخول:
- **البريد**: admin@smartmall.com
- **كلمة المرور**: admin123

---

## 🎯 أول خطواتك في النظام

### 1. تسجيل الدخول
- افتح الرابط: http://localhost
- أدخل بيانات المدير الافتراضية
- اضغط "تسجيل الدخول"

### 2. استكشف لوحة التحكم
- شاهد الإحصائيات الفورية
- تعرّف على الرسوم البيانية
- راجع التنبيهات السريعة

### 3. أضف أول وحدة تجارية
- اذهب إلى "الوحدات التجارية"
- اضغط "إضافة وحدة جديدة"
- املأ البيانات المطلوبة
- اضغط "إضافة"

### 4. أضف مستأجر
- اذهب إلى "المستأجرين"
- اضغط "إضافة مستأجر جديد"
- أدخل بيانات المستأجر
- احفظ البيانات

### 5. أنشئ عقد إيجار
- اذهب إلى "العقود"
- اختر المستأجر والوحدة
- حدد تفاصيل العقد
- أنشئ العقد

## 🔧 الأوامر المفيدة

### إدارة النظام

```bash
# عرض حالة الخدمات
docker-compose ps

# عرض السجلات
docker-compose logs -f

# إعادة تشغيل النظام
docker-compose restart

# إيقاف النظام
docker-compose down

# إعادة بناء النظام
docker-compose build --no-cache
docker-compose up -d
```

### استكشاف الأخطاء

```bash
# فحص سجلات Backend
docker-compose logs backend

# فحص سجلات Frontend
docker-compose logs frontend

# فحص سجلات قاعدة البيانات
docker-compose logs mongodb
```

## 🎨 تخصيص النظام

### تغيير اسم المول

عدّل في ملف `backend/.env`:
```env
APP_NAME=اسم المول الخاص بك
```

### تغيير الألوان

عدّل في ملف `frontend/tailwind.config.js`:
```js
colors: {
  primary: {
    500: '#your-color',
    // ...
  }
}
```

## 📊 نصائح للاستخدام

### ✅ أفضل الممارسات

1. **النسخ الاحتياطي الدوري**
   ```bash
   docker exec smartmall_mongodb mongodump --out /backup
   ```

2. **مراقبة السجلات بانتظام**
   ```bash
   docker-compose logs -f | grep ERROR
   ```

3. **تحديث النظام دورياً**
   ```bash
   git pull
   docker-compose down
   docker-compose build
   docker-compose up -d
   ```

### ⚠️ احذر من

- عدم تغيير بيانات الدخول الافتراضية
- إيقاف النظام أثناء العمليات الحرجة
- حذف البيانات دون نسخ احتياطي
- فتح المنافذ للإنترنت دون حماية
